# dps_main_pipeline

A Snakemake-based  pipeline for comparative protein structure analysis and phylogenetic tree construction.

## Overview

This pipeline integrates a previous structural alignment data with an previous amino acid sequence alignment to create a combined protein alignment and build a phylogenetic tree. It's designed to work with structural alignment files (in `.ali` format, commonly generated by MODELLER) and standard amino acid FASTA alignments.

## Workflow

The pipeline consists of three main steps:
![Pipeline DAG](dag.png)

### 1. **ali_to_fasta** - Structural Alignment Format Conversion
- **Input**: Structural alignment file (`.ali` format)
- **Output**: Structural alignment in FASTA format (`.fasta`)
- **Description**: Converts MODELLER `.ali` format files to standard FASTA format, extracting sequences while removing structure annotations and alignment metadata.

### 2. **profile_alignment** - Profile-Profile Alignment
- **Input**: 
  - Amino acid alignment (`.fasta`)
  - Structural alignment in FASTA format (from step 1)
- **Output**: Combined alignment (`.fasta`)
- **Tool**: [Clustal Omega](http://www.clustal.org/omega/) with `--force` flag
- **Description**: Performs a profile-profile alignment (P1 + P2) to merge the amino acid alignment with the structural alignment into a single, unified alignment.

### 3. **build_tree** - Phylogenetic Tree Construction
- **Input**: Final combined alignment (`.fasta`)
- **Output**: Phylogenetic tree (`.nwk` format - Newick)
- **Tool**: [FastTree](http://www.microbesonline.org/fasttree/) with `-lg` flag
- **Description**: Constructs a maximum-likelihood phylogenetic tree from the combined alignment using the log-gamma correction.

## Setup & Installation

### Requirements
- Conda/Mamba
- Snakemake
- Python 3.13+

## Configuration

Edit `config.yaml` to specify your input and output file paths:

```yaml
# Input files
aa_alignment: "data/aa_aligned.fasta"      # Your amino acid sequence alignment
structural_ali: "data/structural.ali"      # Your structural alignment (MODELLER format)

# Output files
structural_fasta: "results/structural.fasta"    # Converted structural alignment
final_alignment: "results/final_alignment.fasta" # Final combined alignment
tree: "results/tree.nwk"                        # Output phylogenetic tree
```

## Running the Pipeline

### Run with 4 Cores
```bash
snakemake --cores 4
```

### Dry Run (Preview workflow without executing)
```bash
snakemake -n
```

### Generate Workflow Visualization
```bash
snakemake --dag | dot -Tpng > workflow.png
```

## Input Data Requirements

### Amino Acid Alignment (`aa_alignment`)
- Format: FASTA
- Content: Multiple sequence alignment of amino acid sequences
- Example:
  
  ```
  >protein_1
  MKVLWAALLVTFAGCAKAKEVVVIVGPNATGKVALGHIDNVLVPPETPD
  >protein_2
  MKVLWAALLVTFAGCAKAKEVVVIVGPNATGKVALGHIDNVLVPPETPD
  ```

### Structural Alignment (`structural_ali`)
- Format: MODELLER `.ali` format
- Content: Structural alignment with secondary structure annotations
- Typical format markers:
  - Lines starting with `>P1;` indicate sequence headers
  - Lines with asterisks (`*`) mark alignment boundaries
  - `structure:` lines contain secondary structure info (removed in conversion)

## Output Files

After successful execution, check the `results/` directory:

- **`structural.fasta`**: Converted structural alignment in FASTA format
- **`final_alignment.fasta`**: Combined profile alignment (amino acid + structural)
- **`tree.nwk`**: Newick format phylogenetic tree
  - Can be visualized with tools like [FigTree](http://tree.bio.ed.ac.uk/software/figtree/), [PhyloBrowser](http://www.phylobrowser.de/), or online viewers

## Example Workflow

1. **Prepare inputs**: Place your `aa_aligned.fasta` and `structural.ali` files in the `data/` directory
2. **Configure**: Update `config.yaml` with your file paths if needed
3. **Run**: Execute `snakemake --cores all`
4. **Inspect results**: Check the `results/` directory for outputs
5. **Visualize tree**: Open `tree.nwk` in FigTree or similar phylogenetic software

## Troubleshooting

### Clustal Omega errors
- Verify FASTA format of structural.fasta (check for invalid characters)
- Ensure sequences have valid amino acid codes

### FastTree warnings
- Small alignments may produce low confidence values
- Ensure final_alignment.fasta has multiple sequences (>2)

### File not found errors
- Double-check paths in `config.yaml` match your actual file locations
- Use absolute paths if relative paths cause issues
